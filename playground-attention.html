Attention : qui écoute qui ?

Visualise comment un token utilise Q (question) et K (clé) pour décider quels tokens passés écouter, puis collecte leurs V (valeurs).

Nom : Position : Phase :
Prototype 1 — Arcs de connexion. Chaque token est un nœud. Le token sélectionné envoie sa Query (Q) vers les Keys (K) des tokens passés. Le produit scalaire Q·K donne un score. Après softmax, les scores deviennent des poids (épaisseur des arcs). Enfin, les Values (V) sont mélangées selon ces poids.

Phase 1 Q · K → scores

Le token courant projette sa Query. Chaque token passé a une Key. On calcule le produit scalaire Q·K pour mesurer « à quel point ma question correspond à ton étiquette ». Plus le score est élevé, plus le match est fort.

Phase 2 Softmax → poids

Les scores bruts sont transformés en probabilités par softmax : ils totalisent 100 %. C'est ce que tu vois dans la matrice d'attention. L'épaisseur de chaque arc = le poids d'attention.

Phase 3 Poids × V → sortie

Chaque token passé a une Value (son « cahier »). Le token courant fait une moyenne pondérée de toutes les Values, selon les poids d'attention. Le résultat est la sortie de l'attention pour cette position.
